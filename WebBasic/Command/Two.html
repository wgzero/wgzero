# Linux命令

# 1.vim的编写，进入，删除命令

按esc接着操作如下命令： A.:w保存但不退出 B.:wq保存并退出 C.:q退出 D.:q!强制退出，不保存 E.:e!放弃所有修改，从上次保存文件开始编辑命令历史

F.cd 直接回到主目录 G.cd /var/local 去到哪一个目录 H.ls 列举该目录下的文件 I.mkdir创建文件或文件夹 G.mv 改名和移动 K.rm 文件或文件夹的删除

# 2.root权限含义：

A.r是读，w是写，x表示可以执行 B.

# 3.linux下的操作命令：

A.cd 进入一个文件夹 B.ls 列举所有文件夹和文件 C.mkdir 创建文件或文件夹 D.mv 改名和移动 E.rm 文件或文件夹的删除

# 4.文件安装目录和window系统下目录类似

# 5.linux下mysql彻底删除：

A.查看rpm -qa|grey -i mysql B.删除遗留的mysql文件和依赖库 find / -name mysql 查找命令 rm -rf 删除命令 rm -rf /etc/my.cnf 手动删除配置文件

6.node部署：

A.安装node： a.https://blog.csdn.net/u013545439/article/details/97149860

B.pm2：自己开启 npm install -g pm2 全局安装 找不到pm2建立软连接就行了 ln -s /usr/local/nodejs/bin/node /usr/local/bin/ 安装路径

pm2命令： 运行： pm2 pm2 start index.js 运行命令 pm2 stop index.js 关闭

7.apache开启、关闭、重启命令： A.systemctl start httpd 启动 B.systemctl stop httpd 停止服务 C.systemctl restart httpd 重新启动 /usr/local/apache2/htdocs 资源存放

# 8.ngnix安装：

https://www.runoob.com/linux/nginx-install-setup.html ngnix站点：/usr/local/webserver/nginx/html 开启，关闭： /usr/local/webserver/nginx/sbin/nginx 开启 pkill -9 nginx 关闭 /usr/local/webserver/nginx/sbin/nginx -s reload # 重新载入配置文件 /usr/local/webserver/nginx/sbin/nginx -s reopen # 重启 Nginx /usr/local/webserver/nginx/sbin/nginx -s stop # 停止 Nginx

# 9.linux下面部署出现错误，可以先运行node index.js文件

当pm2 start index.js status online 但是 pm2 list status errored 时，运行一下node index.js文件 应该是没有： npm i

# 11.开启防火墙：设置对外开放端口

A.查看防火墙状态:systemctl status firewalld 为dead 为关闭 B.开启防火墙：systemctl start firewalld C.开放默认端口：firewall-cmd --permanent --zone=public --add-port=3306/tcp 为success为成功 D.关闭防火墙：systemctl stop firewalld
